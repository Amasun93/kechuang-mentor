## 2. 数据流:一次完整评估

```
用户填写研究问题
       │
       ▼
[前端] RefineStep 组件
   - 选择比赛(qingchuang/songqingling/chuaying)
   - 点击"让评委老师评估"
       │
       │  POST /api/evaluate
       │  body: { competitionId, profile, project }
       ▼
[后端] routes/evaluate.js
   │
   ├─ 1. 读比赛 JSON → 拿到 evaluation_dimensions + age_group_focus
   │
   ├─ 2. 按 profile.grade 筛 scoring_by_age
   │     (primary → 小学; junior → 初中; senior → 高中)
   │
   ├─ 3. 读 PROMPTX_PERSONA_JSON 环境变量(主人 PromptX 角色)
   │     空时跳过
   │
   ├─ 4. 拼评估 prompt
   │     注入顺序:PromptX → 比赛维度(按学段)→ 联网搜索 → profile
   │
   ├─ 5. 调豆包 chatCompletion
   │     │
   │     ├─ 配 KEY → 真实豆包 → 解析 JSON 输出
   │     │
   │     └─ 未配 KEY → fallbackEvaluate(规则降级)
   │                  基于维度字段填充情况生成反馈
   │
   └─ 6. 构造响应(只返回孩子端字段)
       │
       │  { studentFeedback, suggestedQuestions, strongPoints, mock }
       ▼
[前端] RefineStep 显示评估结果
   - 评委老师的话(3 段)
   - 评委可能问的 3 个问题
   - 你做得不错的地方
   │
   └─❌ 不显示具体分数/等级
```

## 3. 数据流:AI 对话(苏格拉底式)

```
用户输入
   │
   ▼
[前端] AIAssistant
   - 加载当前 step 的 history(localStorage)
   - 拼 system prompt:
     buildFullSystemPrompt({
       step: 'structure',
       personalityPrompt: '暖心学姐的 personaPrompt',
       profile: { grade: 'junior', interests: [...] }
     })
   - 系统会按顺序拼接:
     ① SYSTEM_PROMPT(苏格拉底哲学)
     ② personalityPrompt(性格侧重)
     ③ stagePrompt + agePrompt(步骤 + 年级)
     ④ profileBlock(学生档案)
   │
   │  POST /api/ai/chat
   │  body: { system, messages }
   ▼
[后端] routes/ai.js
   │
   ├─ 配 KEY → 调豆包 → 返回
   │
   └─ 未配 KEY → 走 mock 库(根据 step + persona 生成)
   │
   ▼
[前端] 显示回复 + 写入 history
```

## 4. 关键技术决策

| 决策 | 理由 |
|------|------|
| **不 fork 参考站** | 完全自主实现,代码、prompt、数据均为原创 |
| **localStorage 优先** | MVP 阶段避免后端存储,V0.1 再切 |
| **后端代理豆包** | 不暴露 API Key + 统一 mock 降级 |
| **Prompt 可配置** | 评估路由预留 PROMPTX_PERSONA_JSON 接口 |
| **Tailwind 暗金主题** | 用主题色而非内联,支持性格/状态变色 |
| **步骤独立 session** | 6 步流程每个步骤独立保存 AI 对话 |
| **浮动窗 React state** | 不用第三方拖动库,自写 50 行 |
| **PDF 浏览器端生成** | html2pdf.js,无后端依赖 |
| **评估路由规则降级** | 没配豆包 KEY 时也能跑出有效反馈 |
| **案例按学段过滤** | 小学生不推荐高中生课题,反之亦然 |

## 6. 安全与限制

- **API Key**:只在后端,前端不暴露
- **CORS**:默认 `localhost:5173`,生产用具体域名
- **学生数据**:MVP 用 localStorage,V0.1+ 走后端加密存储
- **PromptX 角色**:从环境变量读,不持久化
- **评估不暴露评分**:前端组件严格只渲染 `studentFeedback/suggestedQuestions/strongPoints`

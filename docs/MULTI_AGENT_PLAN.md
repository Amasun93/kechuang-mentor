## 4. 协调机制

### 4.1 中央文档（PROJECT_BRIEF.md）

**唯一契约源**。所有 Agent 开始工作前必须先读这份文档，理解产品定位、6 步流程、苏格拉底哲学、验收标准。

### 4.2 进度同步协议

- Agent 完成里程碑 → 回主对话报告
- Agent 跑挂 / 卡住 → 自动推送
- Agent 之间通过主对话转发（不直接互发）

### 4.3 文档变更协议

如果 Agent 发现需要改中央契约（PROJECT_BRIEF.md）：
1. 不能直接改
2. 在报告里提"建议变更 X"
3. 主对话评估后决定
4. 主对话统一更新文档

### 4.4 冲突解决

如果两个 Agent 报告矛盾：
- 主对话以 PROJECT_BRIEF.md 当前内容为准
- 必要时回主人决策
